# Dashboard V2 设计规格

## 目标

将 A股全景仪表盘从"并列指标报告"升级为"带决策推理+持仓追踪+回测复盘的学习型交易系统"。

## 约束

- 不动现有10个指标分析层的输出结构
- 所有计算必须精确，前后天数据必须连续（不能出现计算脱节）
- 每天自动运行，飞书推送

## 新增模块

### 1. ⚔️ 信号冲突裁决

**触发条件**：任意两个指标 status 不同时（一个 healthy 一个 danger，或一个 healthy 一个 caution）。

**优先级（来自《趋势交易论》框架权重）**：
1. 三周期趋势（最高）——月线方向压倒一切
2. 量价结构 —— 放量滞涨 > 一切看多信号
3. 520战法 —— 死叉 > RSI超卖
4. MACD动能 / RSI / 乖离率 / 年线 / 支撑压力 / 波动率 / M1（最低）

**裁决输出格式**：对每对冲突，说明(A)谁压倒谁、(B)为什么、(C)实际该怎么做。

**无冲突时**：输出 "今日指标一致，无冲突" + 一致性方向（偏多/偏空/中性）。

### 2. 🔍 板块操作信号

取代旧版"板块全貌"中模糊的"关注/注意"标签。每条输出是：具体交易条件 + 具体动作 + 止损/止盈价位。

**分类**：
- 🟢 可考虑买入：底部企稳+技术面转好 → 条件X满足→做Y，止损Z
- 🔴 应考虑减仓/清仓：高位+量价背离 → 具体减仓比例+清仓条件
- 🟡 继续等待：条件不满足 → 具体等待什么信号

每个建议附带理由引用（《趋势交易论》哪一节）。

### 3. 💰 模拟账户

**初始状态**：现金 1,000,000，起始日期为首次运行日。

**数据文件 `portfolio.json`**：
```json
{
  "start_date": "2026-07-24",
  "initial_cash": 1000000,
  "cash": 720000,
  "holdings": {
    "半导体ETF": {
      "shares": 14000,
      "avg_cost": 10.0,
      "current_price": 9.43,
      "market_value": 132020,
      "pnl_pct": -5.7,
      "entry_date": "2026-06-28",
      "entry_reason": "底分型+站上5日线"
    }
  },
  "total_market_value": 1005000,
  "cumulative_return_pct": 0.5,
  "last_update": "2026-07-24"
}
```

**每天操作逻辑**：
1. 读上次 portfolio.json
2. 更新持仓市值（用当日收盘价）
3. 基于当日信号生成买卖建议
4. 如果用户接受建议（手动确认），执行交易，更新 portfolio.json
5. 如果是自动模式（GitHub Actions），记录建议但不执行，次日核对

**关键：计算精度**
- 所有金额精确到元
- 股数精确到整数（A股最小100股/手）
- 收益率精确到0.1%
- 每步计算前校验：前日持仓市值 + 前日现金 = 前日总资产。今日操作后的 持仓市值 + 剩余现金 = 今日总资产。不相等则报错。

### 4. 📋 建议连续性

**数据文件 `position_state.json`**：
```json
{
  "last_date": "2026-07-23",
  "last_index_recommendation": "空仓或轻仓(0-20%)",
  "last_sector_recommendations": {
    "半导体": {"action": "减仓一半", "date": "2026-07-22"},
    "仪器仪表": {"action": "等站上20日线入场", "date": "2026-07-22"}
  }
}
```

每天仪表盘先输出上次建议回顾，再给本次建议。如果上次建议已执行，本次基于新状态给建议。

### 5. 📊 月度回测复盘

**数据文件 `trade_log.jsonl`**：每次运行追加一行 JSON。
```json
{"date":"2026-07-24","type":"index","target":"上证","signal":"三周期向下","recommendation":"空仓","indicators":{...}}
{"date":"2026-07-24","type":"sector","target":"半导体","signal":"顶分型+放量跌","recommendation":"减仓一半","entry_price":9.43}
```

**脚本 `backtest_review.py`**：
- 每月1号自动运行（或手动触发）
- 回看30天内的每条建议
- 对比实际走势（从AKShare取后续N天K线）
- 输出：信号正确率、账户表现 vs 基准、发现的问题、改进建议

**正确率判定**：
- 买入信号：后续5/10/20天涨幅 > 0 → 正确
- 卖出信号：后续5/10/20天跌幅 > 0 → 正确
- 仓位建议：与大盘方向一致 → 正确

## 非功能要求

- **数值一致性**：每次运行严格校验 `前日总资产 == 现金 + sum(各持仓市值)`，`操作后现金 >= 0`
- **幂等性**：同一天重复运行不重复记录交易日志（检查 date 是否已存在）
- **容错**：若 portfolio.json 损坏或缺失，从初始100万现金重新开始，并在输出中标注 ⚠️
- **飞书推送**：保持现有推送逻辑不变，内容超长时分多条发送
